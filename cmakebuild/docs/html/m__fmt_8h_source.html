116 /*! \addtogroup m_fmt Format String Functions
117  * \ingroup m_string
118  *
119  * Formatted String output.
120  *
121  * %<character> is used to denote the data type of the argument. Arguments are passed
122  * after the format string.
123  *
124  * # Supported features
125  *
126  * ## Flags
127  *
128  * Flag | Description
129  * -----|------------
130  * '-' | Left justify output. Default is to right justify. Overrides the '0' flag if both are set.
131  * '+' | Always add the sign for numeric output. Default is only to add sign for negative. Overrides the ' ' flag if both are set.
132  * '#' | Add the appropriate prefix to the output of numerics. 0x or 0X for Hex. 0 for Octal.
133  * ' ' | Use a space as if it were the sign for positive numbers.
134  * '0' | Pad numerics with 0. Default padding is space (' ').
135  *
136  * ## Width and precision
137  *
138  * A decimal (.) separated value with each side is option can be specified, <width>.<precision>.
139  *
140  * The width is the minimum output size. Padding will be added if the output would be smaller
141  * than the width. If the output size exceeds the width the width is ignored and the full output
142  * will be used.
143  *
144  * Precision for strings is how much to read of the string. If the value is larger than the length
145  * of the string, the string length will be used. E.g. ("%.2s", "abc") will result in "ab" for the output.
146  *
147  * Precision for floating point determines the number of decimal places to output. The default is 6.
148  * It's recommended the maximum precision specified be no large than 14 digits. Digits over 14 can have
149  * platform specific rounding differences.
150  *
151  * If either or both sides of the decimal do not have a value the side is ignored. This allows any
152  * combination of width and precision to be specified.
153  *
154  * A '*' can be specified for either which will read the value from an argument. The argument is an
155  * int. The arguments are read right to left. E.g. ("%*.*s", 4, 2, "abc") will result in " ab".
156  *
157  * ## Size modifiers
158  *
159  * Specify the data size of a given argument.
160  *
161  * Modifier | Description
162  * ---------|------------
163  * hh | Size of char. 8 bit.
164  * h | Size of short. 16 bit.
165  * l | Size of long. 8 or 16 bit (system dependant).
166  * ll | Size of long long. 64 bit.
167  * I, z | size of size_t. Based on system size. 32 or 64 bit.
168  * I64 | 64 bit.
169  * I32 | 32 bit.
170  *
171  * ## Conversion
172  *
173  * Specifies the data type of the argument.
174  *
175  * Type | Description
176  * -----------------|------------
177  * d, i | Signed integer.
178  * o, O | Unsigned integer. Output as octal.
179  * u | Unsigned integer.
180  * x, X | Unsigned integer. Output as hex. 'x' outputs lowercase, 'X' outputs uppercase.
181  * P, P | Unsigned pointer. Output as hex. 'x' outputs lowercase, 'X' outputs uppercase.
182  * e, E, f, F, g, G | Double. All will output in the form [-]ddd.ddd. Default 6 decimal digits unless otherwise precision is otherwise specified.
183  * c | signed character.
184  * s | Strong (const char *).
185  *
186  * @{
187  */
